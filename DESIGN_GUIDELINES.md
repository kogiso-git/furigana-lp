## 1. Brand Essence / ブランドの核

### FURIGANA.とは何か

FURIGANA.は、社長の頭の中にある判断基準・価値観・現場感覚・暗黙知を、
**言葉・仕組み・AI活用に翻訳し、会社で使える形、また社長自身が使える形に整える**
個人ブランドサービスである。

提供者は「社長の翻訳者」という肩書きを持つ、小木曽秀弥。

### 「社長の翻訳者」とは何か

- 社長の代わりに仕事をする「代行業者」ではない
- スキルを教える「AIスクール講師」でもない
- 社長の言語化できていない判断基準を引き出し、会社や社長自身が使える形に変える「翻訳者」
- 最終的に、社長自身が自分の手で仕事を動かせる状態を目指す

### キーメッセージの扱い

```
社長の当たり前に、ふりがなを振る。
```

- これはブランドの核であり、変更不可
- スマホで中途半端に折れさせない（`<span class="hero-line">` で改行制御）
- 縦のリズムで読む：「社長の当たり前に、」→「ふりがなを振る。」
- 「当たり前」という言葉に全てが宿っている。社長には当たり前でも、周囲には伝わっていない

### AIを主役にしない理由

- AIはあくまで道具。主役は「社長の判断」と「会社の資産」
- 「AI活用支援」と打ち出すと AIスクール感・SaaS感が出る
- FURIGANA.の価値は「翻訳」であり、道具の選択は後段
- コピー・見出しで「AI」「ChatGPT」「Claude」が前面に出すぎないよう注意

### 代行ではなく「社長自身が動かせるようにする」思想

- こちらが全部作って渡すことが目的ではない
- 社長自身が、仕事を言葉にし、仕組みにし、動かせるようになること
- この思想をコピー・CTA・サービス説明の全箇所に一貫させる

## 2. Visual Direction / 目指す見え方

FURIGANA.のビジュアルは、以下の言葉で形容される。

- **銀座・青山系** — 過剰に主張せず、静かに品格を持つ
- **和モダン** — 明朝体、余白、罫線が作る日本の紙面感
- **ZINE編集感** — 章番号・英字ラベル・段組みで作るエディトリアルな質感
- **高級ブランドブック** — 情報が多すぎず、余白が高級感を作る
- **余白** — 余白は「何もない場所」ではなく、「品格を作る空間」
- **静けさ** — 何かを大声で主張しない。読む人が自分から近づく設計
- **紙面感** — スクロールするWebページではなく、めくるような読書体験
- **明朝体** — 見出しはNoto Serif JP。ゴシックに頼らない
- **上品なグレー** — チャコール〜ストーンのグレーレンジで整理する
- **少量のネイビー** — 署名・プロフィール名など、ここぞの場面だけに使う
- **実務に強いが、売り込まない** — 事例・数字は具体的に。でも押しつけない

## 3. What to Avoid / 避ける見え方

以下はFURIGANA.のビジュアルから最も遠い印象。これに近づいたら即見直す。

### スタイル面
- ❌ AIスクール感（ChatGPT映えする青いUI、学習ステップ表示）
- ❌ 情報商材LP感（赤背景、爆速、誰でも、稼げる、無料強調、煽り見出し）
- ❌ SaaSテンプレ感（ヒーロー背景グラデ、機能ブロック、価格比較表）
- ❌ 派手なグラデーション・強いグロー・ネオン
- ❌ 強い影（`box-shadow` の多用）
- ❌ 強い角丸（`border-radius: 12px` 以上の多用）
- ❌ ボタンだらけの画面（CTAが3つ以上並ぶ）
- ❌ カードUIだらけ（囲い・背景色・影で情報をパッケージングしすぎる）
- ❌ 文字を大きくして「目立たせるだけ」の設計
- ❌ Webflowテンプレ風のアニメーション（スライドイン多用など）
- ❌ スタートアップ風の青い未来感

### コピー面
- ❌ 爆速・誰でも簡単・最新AI・AIをマスター・完全自動化
- ❌ 稼げる・成功保証・年収が上がる
- ❌ DX推進だけを前面に出す
- ❌ 無料の過剰強調（FREE連打など）

## 4. Color System / 色の使い方

現在のCSS変数：

| 変数名 | 用途 |
|---|---|
| `--c-bg` `#F7F5F0` | ページ背景（生成り紙色） |
| `--c-cream` `#ECE8DE` | セクション背景・プレースホルダー |
| `--c-card` `#FFFDF8` | テーブル行・カード背景 |
| `--c-ink` `#171C1B` | 最暗色。強調見出し・重要テキスト |
| `--c-charcoal` `#2F3331` | 本文・CTA・ボタン塗り |
| `--c-gray` `#595E5C` | 補足テキスト・セクションラベル |
| `--c-stone` `#A7A39A` | 装飾・アクセント数字 |
| `--c-border` `#CCC7BC` | 罫線 |
| `--c-navy` `#1A2A44` | 署名色（プロフィール名など少量） |
| `--c-dark` `#222522` | フッター・最終CTA背景 |

### カラー運用ルール

- **色数は増やさない** — 上記変数以外の色を追加しない
- 目立たせるために新しい色を足さない（余白・罫線・フォントウェイトで対応）
- CTAはチャコール（`--c-charcoal`）または墨（`--c-ink`）を軸にする
- アクセントカラーは不要。FURIGANA.の強みは色ではなく余白と文字
- ダーク背景（フッター・最終CTA）はネイビーではなく `--c-dark`（ほぼ黒）で統一

## 5. Typography / タイポグラフィ

### フォント体系

| 変数名 | フォント | 用途 |
|---|---|---|
| `--f-en` | Cormorant Garamond | 英字ラベル・装飾・数字 |
| `--f-ja-serif` | Noto Serif JP | 日本語見出し（セクション・カード） |
| `--f-ja-sans` | Noto Sans JP | 日本語本文・UIテキスト |

### 数字の扱い

- `font-variant-numeric: lining-nums` を `body` に設定済み（オールドスタイル数字の飛び出し防止）
- 料金表には追加で `tabular-nums` を適用する
- 数字は「装飾」ではなく「情報の階層」として機能させる
- 大きな装飾数字は使いすぎない

### タイポグラフィルール

- **見出し**は明朝体（`--f-ja-serif`）を軸にする
- **本文**はゴシック（`--f-ja-sans`）で読みやすさを優先
- **英字ラベル**はCormorant Garamondを使うが、`0.75rem`以下にしない
- 太字（`font-weight: 700`以上）は原則使わない
- 文字サイズを上げるだけで問題を解決しない
- **行間・文字間・余白で上品に整える**
- `letter-spacing` は英字ラベルに多め、日本語本文は控えめに

### Heroコピーの改行ルール

```html
<!-- ✅ 正しい実装 -->
<h1 class="hero-heading">
  <span class="hero-line">社長の当たり前に、</span>
  <span class="hero-line">ふりがなを振る。</span>
</h1>
```

- スマホ375pxで「社長の当たり前に、」と「ふりがなを振る。」が確実に2行で収まるよう `display: block` で制御
- 自然改行に任せない。中途半端な折れは許容しない
- フォントサイズは `clamp(2.1rem, 9.5vw, 3.2rem)` が現在の基準

## 6. Layout / 余白・罫線・紙面構成

### 余白の思想

- 余白は「何もない場所」ではなく**「品格を作る空間」**
- PCでは余白を活かして高級感を出す
- スマホでは広げすぎず、読み進めやすいリズムを作る
- セクション間には呼吸を入れる（詰め込まない）

### 罫線の使い方

- カードUIの代わりに `border-top` などの罫線で情報を整理する
- 強い囲み・強いカード（背景色＋影）を避ける
- `--c-border`（`#CCC7BC`）の1px罫線を基本にする

### 段組みとZINE編集感

- 英字ラベル（LISTEN / STRUCTURE / ACCOMPANY）＋日本語見出しの組み合わせで編集感を作る
- 段組み（CSS Grid）は `border-top` の横罫線で区切る
- 数字の装飾多用はテンプレ感につながる。英字ラベルと見出しを主役にする

### セクションの基本構造

```
[section-label]    ← 小さな英字ラベル（TRANSLATE / FOR YOU 等）
[section-heading]  ← 日本語見出し（明朝体・主役）
[本文 / ブロック]
```

## 7. Header Navigation / ヘッダーナビ

### PC / タブレット（768px以上）

表示する項目：
- 支援内容 → `#services`（ページ内リンク）
- プロフィール → `#profile`（ページ内リンク）
- SNS → `#sns`（ページ内リンク）
- 無料相談 → GoogleフォームURL（外部リンク）

### スマホ（767px以下）

- 無料相談ボタンのみ表示
- 支援内容 / プロフィール / SNS は非表示（`display: none`）
- ヘッダーが窮屈にならないようにする
- フッターでページ回遊の補完をしてよい

### リンクルール

- ページ内リンクには `target="_blank"` を付けない
- 外部リンク（Googleフォーム等）には必ず `target="_blank" rel="noopener noreferrer"` を付ける
- 無料相談のURLは全箇所で同一URLに統一する
- `YOUR_GOOGLE_FORM_URL` などの仮URLを残さない

## 8. CTA / 無料相談導線

### CTAの温度感

- CTAは売り込みではなく、**招待状のように扱う**
- 「申し込む」より「はじめる」「相談する」の温度感
- 「無料！今すぐ！」の過剰強調を避ける

### CTAの配置ルール

- 配置箇所：Hero / 進め方セクション後 / 最終CTA の3箇所
- それ以上むやみに増やさない
- Hero内のghostボタン（「サービスを見る →」）はスマホではテキストリンク化してもよい

### スマホでのCTA

- CTAボタンが大きすぎてHeroを圧迫しないようにする
- ボタン幅は`width: 100%`にしない（画面幅いっぱいのCTAはLP安っぽさが出る）
- `padding` と `font-size` は適度に抑える

## 9. Mobile First Rules / スマホ表示の優先ルール

### 大前提

> スマホはPCの縮小ではない。
> 名刺交換後、その場でスマホで開かれる前提で設計する。

### 確認基準

- **375px幅を最優先で確認する**（iPhone標準サイズ）
- 320pxでも破綻しない
- `@media (max-width: 767px)` で必ず上書きする

### スマホでの優先事項

- Heroの改行制御を最優先で確認する
- ヘッダーはロゴが潰れない高さを保つ（現在64px）
- ナビよりCTAを優先する
- 画像・`service-fig` の高さが本文を圧迫しない（`max-height: 220px` 上限）
- セクション見出し（`.section-heading`）の不自然な改行を避ける（現在`clamp(1.45rem, 6.5vw, 2.05rem)`）
- スクロール疲れを起こさない（セクション数が多いため間延び注意）

### スマホNG事項

- ❌ Heroコピーが3行以上に折れる
- ❌ ヘッダーにロゴ＋ナビが詰まって窮屈
- ❌ 画像が縦長すぎてスクロールが重い
- ❌ 見出しが1文字だけ次行に落ちる
- ❌ ボタンが画面幅いっぱいに広がりすぎる

## 10. Images / 画像の扱い

### 画像の位置づけ

- 写真は主役ではなく、**紙面に差し込む図版**として扱う
- 強い角丸・強い影・派手な写真ギャラリーにしない

### サービス画像（service-fig）

- 比率：PC = `aspect-ratio: 4/3`、スマホ = `aspect-ratio: 16/10`（`max-height: 220px`）
- 画像未設定時はクリーム背景の `::after "Photo"` ラベルで未完成感を出さない
- 画像を入れたとき：`<img>` を `.service-fig` 内に追加するだけで差し替え完了
- `:has(img)::after { display: none }` により "Photo" ラベルは自動消える

### ファイル命名規則

```
assets/images/
  service-listen.jpg
  service-structure.jpg
  service-accompany.jpg
  profile.jpg
  hero.jpg
  ogp.jpg  （1200×630px）
```

## 11. Copywriting Tone / コピーの書き方

### 基本姿勢

- 短く、深く、伝わる
- 説明しすぎない
- 社長の現実に寄り添う
- 専門用語を使いすぎない。ただし安っぽくしない
- AIを主語にしすぎない

### 軸にする言葉

- 「社長の判断」
- 「会社の資産」
- 「社長自身の手で仕事を動かす」
- 「社長の当たり前を、言葉にする」

### 避ける表現

| ❌ 避ける | ✅ 代わりに |
|---|---|
| 爆速・誰でも簡単 | 一緒に整理する・少しずつ整える |
| 最新AI・AI完全活用 | 必要なところにだけ、道具として使う |
| 完全自動化 | 社長自身が動かせる仕組みに |
| 稼げる・成功保証 | 判断の精度が上がる・説明が減る |
| 申し込む・今すぐ | はじめる・相談する |

## 12. Component Rules / UI部品ルール

### ボタン

- Primary（塗り）: `background: --c-charcoal`, `border-radius: 2px`
- Ghost（枠のみ）: `border: 1px solid`, 背景なし
- スマホではghostボタンをテキストリンク化してもよい
- ❌ `border-radius: 8px` 以上の丸ボタン
- ❌ グラデーション背景ボタン
- ❌ 影付きボタン

### テキストリンク

- ホバー: 色変化 or 薄い下線（`border-bottom: 1px solid`）
- ❌ 下線を常時表示するのはカジュアルすぎる
- ❌ 青い標準リンク色は使わない

### セクションラベル

- Cormorant Garamond, italic, letter-spacing 広め
- `color: --c-gray`（主役ではなく編集記号）
- サイズは `0.78〜0.84rem`（小さすぎない）
- 章番号（01 / など）は原則不要。英字キーワードだけで十分

### サービスブロック（service-card）

- 大きな装飾番号（`service-num`）は使わない
- 英字ラベル（LISTEN / STRUCTURE / ACCOMPANY）を編集的な小見出しとして配置
- 強いカード囲みを避ける（背景色・影・大きな角丸なし）
- 画像 → 見出し → 本文 → タグの順

### 進め方ブロック（how-block）

- PC: `grid-template-columns: 200px 1fr`（左：サービス名、右：ステップ）
- スマホ: 1カラム
- `border-top` の罫線で区切る（カード化しない）
- サービス名ラベル（LISTEN / STRUCTURE / ACCOMPANY）は `how-block-label` で静かに
- ステップ番号は `STEP 1` 形式か、シンプルな `01` を小さく薄く

### 料金表

- テーブルは `border-top` の罫線で区切る
- `pricing-price` には `tabular-nums` で桁を揃える
- FREE バッジのみ `--c-charcoal` 塗り（他のバッジを増やさない）
- ❌ 価格をハイライトボックスで囲う
- ❌ 「人気！」「おすすめ」バッジの乱用

### FAQ

- アコーディオン形式（script.js で制御済み）
- 開閉は `aria-expanded` で管理
- デザインは罫線のみ。カード化しない

### SNSリンク（セクション内）

- ピル型ボタン：`border: 1px solid --c-border`, `border-radius: 50px`
- ホバー: ボーダーが `--c-charcoal` に濃くなる
- SNSアイコンは現状なし（英字テキストのみ）

### フッター

- 背景: `--c-dark`（ほぼ黒）
- SNSリンク: ピル型（ダーク背景用：白系低透明度ボーダー）
- シンプルに：ブランド名・タグライン・SNSリンク・コピーライトのみ
- フッターナビは原則不要（ヘッダーナビが機能していれば）

## 13. Accessibility / アクセシビリティ

- 文字色は薄くしすぎない（`--c-stone` 以上の明度は補足用途のみ）
- 英字ラベルは `0.75rem` 以下にしない
- CTAボタンのタップ領域は `min-height: 44px` を確保する
- 画像には `alt` テキストを入れる（装飾画像は `aria-hidden="true"`）
- `<a href="#main-content">本文へスキップ</a>` のスキップリンクを維持する
- キーボード操作対応: `a:focus-visible`, `button:focus-visible` のスタイルを維持する
- 外部リンクには必ず `target="_blank" rel="noopener noreferrer"` を付ける

## 14. SEO / OGP / 公開前チェック

```html
<!-- 公開前に必ず差し替えるもの -->
<meta name="description" content="..."> ← 内容を確認
<meta property="og:url" content="https://example.com"> ← 公開URLに変更
<meta property="og:image" content="assets/images/ogp.jpg"> ← 実在ファイルか確認
```

### 公開前チェックリスト

- [ ] `YOUR_GOOGLE_FORM_URL` が残っていない
- [ ] GoogleフォームURLが全CTAで統一されている
- [ ] `og:url` が実際の公開URLになっている
- [ ] `ogp.jpg`（1200×630px）が存在する
- [ ] ブラウザのコンソールにエラーが出ていない
- [ ] スマホ実機（または375px）で確認済み
- [ ] `title` と `description` が内容と合致している

```
□ 情報商材LPっぽくなっていない
□ script.js が意図通りに動いている（フェードイン・FAQ・スクロール）
□ console エラーが出ていない
```

## 17. Decision Principles / 判断に迷った時の原則

> どう実装すればいいか迷ったとき、以下の原則に立ち返ってください。

- **迷ったら、派手にせず静かに整える。**
- **迷ったら、色ではなく余白と罫線で整理する。**
- **迷ったら、AIではなく社長の判断を主語にする。**
- **迷ったら、ボタンを増やさず、導線を整理する。**
- **迷ったら、PCより先にスマホで確認する。**
- **迷ったら、説明を増やさず、見出しを磨く。**
- **迷ったら、「社長の当たり前に、ふりがなを振る。」に戻る。**
